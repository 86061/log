Create Triggered After Posting To Automate All Rituals .js

Master
Category :development Tags: automation netlify Date: 12/18/2024, 9:46:11 PM

Kita akan mengotomatisasi serangkain ritual yang harus dilakukan setelah mengirim data posting ke form di netlify.

Ritual saya setelah membuat sebuah post artikel baru adalah:

membuka https://postnetlify.netlify.app/saveFormsToStatic dahulu untuk membuat versi file static dari posts nya,

menunggu 4 menit,

lalu membuka https://postnetlify.netlify.app/staticIndexMaker untuk membuat index pages files versi static file nya , 

menunggu 3 menit,

lalu membuka https://postnetlify.netlify.app/rssfeed ,

menunggu 1 menit , 

lalu membuka https://postnetlify.netlify.app/sitemap-xml

, dan ditunggu 1 menit.

 

Namun, kita bisa menggunakan pendekatan alternatif agar otomatisasi tetap berjalan saat data form baru dikirim ke Netlify.

Berikut adalah pendekatan yang bisa digunakan:

1. Gunakan Netlify Webhooks untuk Memicu Fungsi

Netlify menyediakan webhooks yang memungkinkan kamu untuk memicu tindakan tertentu saat ada form yang dikirim. Jadi, kita bisa menghubungkan Netlify Form dengan Netlify Functions menggunakan webhook.

2. Langkah-langkah:

a. Buat Fungsi di Netlify

Seperti yang sudah dibahas sebelumnya, kamu perlu membuat fungsi di Netlify. Fungsi ini akan menjalankan proses otomatisasi yang sama dengan yang kamu inginkan (mengakses URL tertentu dan menunggu beberapa detik).

Buat fungsi file di /netlify/functions/triggered-after-posting-to-automate-all-rituals.js seperti berikut:

	const fetch = require('node-fetch');

exports.handler = async function(event, context) {
  // Step 1: Create static version of the post
  await fetch('https://postnetlify.netlify.app/saveFormsToStatic');
  
  // Step 2: Wait for 4 minutes
  await new Promise(resolve => setTimeout(resolve, 240000));

  // Step 3: Create static index page files
  await fetch('https://postnetlify.netlify.app/staticIndexMaker');
  
  // Step 4: Wait for 3 minutes
  await new Promise(resolve => setTimeout(resolve, 180000));

  // Step 5: Update RSS feed
  await fetch('https://postnetlify.netlify.app/rssfeed');
  
  // Step 6: Wait for 1 minute
  await new Promise(resolve => setTimeout(resolve, 60000));

  // Step 7: Update sitemap
  await fetch('https://postnetlify.netlify.app/sitemap-xml');
  
  // Step 8: Wait for 1 minute
  await new Promise(resolve => setTimeout(resolve, 60000));

  return {
    statusCode: 200,
    body: JSON.stringify({ message: 'Automation completed successfully.' })
  };
};

b. Set Webhook di Netlify

  1. Masuk ke Netlify Dashboard.
  2. Pilih situs yang kamu ingin atur.
  3. Klik pada menu Site settings.
  4. Di bawah Build & deploy, pilih Build hooks.
  5. Klik Add build hook, beri nama, dan pilih Deploy Settings (misalnya: "Trigger post automation").
  6. Setelah itu, kamu akan mendapatkan URL webhook yang bisa digunakan untuk memicu tindakan tertentu (misalnya fungsi automatePost).

c. Buat Webhook di Netlify Forms

Setelah kamu membuat webhook di Netlify, kamu perlu menambahkan webhook tersebut ke form kamu agar proses ini dijalankan ketika form di-submit.

  1. Pergi ke Forms di Netlify dashboard.
  2. Pilih form yang ingin kamu tetapkan webhook-nya.
  3. Tambahkan webhook yang baru saja dibuat ke bagian Webhook di pengaturan form.

Setiap kali ada form yang dikirim, webhook ini akan dipicu, dan Netlify akan menjalankan fungsi yang telah kita buat (yaitu fungsi yang mengakses URL dan menunggu sesuai dengan langkah-langkah yang kamu inginkan).

3. Verifikasi

Setelah setup ini selesai, kirimkan form melalui Netlify Forms dan pastikan webhook terpicu dan fungsi berjalan sesuai harapan. Kamu bisa memeriksa log di Netlify Functions untuk memverifikasi apakah setiap langkah otomatisasi berjalan dengan benar.

Dengan metode ini, kamu tidak perlu mengandalkan GitHub Actions, karena proses ini akan sepenuhnya ter-trigger oleh form submission di Netlify. Jika ada yang perlu diperjelas, beri tahu saya!
